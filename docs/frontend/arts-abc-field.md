# Новое поле `abc` в модели Art

## Что изменилось

В модель артикула (`Art`) добавлено новое опциональное строковое поле `abc`. Поле предназначено для ABC-классификации товара.

## Затронутые эндпоинты

### POST `/api/arts/upsert` (создание/обновление)

Теперь в объектах массива body можно передавать поле `abc`.

**Пример запроса:**

```json
[
  {
    "artikul": "12345",
    "zone": "A1",
    "nameukr": "Назва товару",
    "namerus": "Название товара",
    "limit": 50,
    "abc": "A"
  }
]
```

Поле опциональное — если не передано, в документе будет `undefined`.

### GET-эндпоинты (чтение)

Все GET-эндпоинты, возвращающие объекты Art, теперь включают поле `abc` в ответе:

- `GET /api/arts` — список с пагинацией
- `GET /api/arts/id/:id` — по ID
- `GET /api/arts/artikul/:artikul` — по артикулу
- `GET /api/arts/zone/:zone` — по зоне

**Пример объекта Art в ответе:**

```json
{
  "_id": "...",
  "artikul": "12345",
  "nameukr": "Назва товару",
  "namerus": "Название товара",
  "zone": "A1",
  "limit": 50,
  "marker": "20260314",
  "abc": "A",
  "btradeStock": {
    "value": 75,
    "date": "2026-03-14T10:00:00.000Z"
  },
  "createdAt": "2026-03-14T08:00:00.000Z",
  "updatedAt": "2026-03-14T10:00:00.000Z"
}
```

### GET `/api/arts/export` и GET `/api/arts/export-with-stocks` (Excel-экспорт)

Excel-файлы теперь содержат колонку **ABC**.

## Формат данных поля

| Свойство | Значение |
|----------|----------|
| Имя поля | `abc` |
| Тип | `string` |
| Обязательность | Опциональное |
| Передается при upsert | Да |
| Присутствует в ответах | Да (если задано) |
| Колонка в Excel | ABC |

## Рекомендации для фронтенда

1. При отображении списка артикулов — добавить колонку ABC в таблицу.
2. При создании/редактировании артикула в форме upsert — добавить текстовое поле `abc`.
3. Поле не обязательное, форма должна корректно работать без его заполнения.
4. В Excel-экспортах колонка ABC расположена после колонки Маркер.
