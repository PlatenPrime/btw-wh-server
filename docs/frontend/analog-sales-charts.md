# Эндпоинты продаж по аналогу для графиков (sales-by-date, sales-range)

## Назначение

Два эндпоинта возвращают данные для построения графиков **продаж** и **выручки** по одному аналогу: на одну дату и за период. В отличие от эндпоинтов срезов (`/analog/:analogId` и `/analog/:analogId/range`), здесь в ответе не остатки (stock), а объём продаж за день (разница остатка с предыдущим днём) и выручка (продажи × цена). Логика расчёта совпадает с той, что используется в Excel-отчёте сравнения продаж.

## Эндпоинты

### GET `/api/analog-slices/analog/:analogId/sales-by-date`

Одна точка данных на запрошенную дату.

**Запрос:** path `analogId` (MongoDB ObjectId аналога), query `date` (YYYY-MM-DD).

**Ответ 200:** JSON с полем `data`: объект `{ sales, revenue, price, isDeliveryDay }`. `sales` — объём продаж в штуках за этот день; `revenue` — выручка (sales × price); `price` — цена на дату; `isDeliveryDay` — true, если в этот день остаток вырос относительно предыдущего (день поставки, продажи считаются 0).

**Ошибки:** 400 — невалидный analogId или date; 404 — аналог не найден, у аналога пустой artikul или нет среза на эту дату. Тело ошибки: `{ message: string }` или `{ message, errors }` при 400.

### GET `/api/analog-slices/analog/:analogId/sales-range`

Массив точек за период для построения графиков по осям дат.

**Запрос:** path `analogId`, query `dateFrom` и `dateTo` (YYYY-MM-DD), dateFrom ≤ dateTo.

**Ответ 200:** JSON с полем `data`: массив объектов `{ date, sales, revenue, price, isDeliveryDay }`. `date` — строка в формате ISO (например для оси X). Массив отсортирован по дате по возрастанию. В массив входят только те даты, по которым есть срез и запись по артикулу аналога.

**Ошибки:** 400 — невалидный analogId или даты (в т.ч. dateFrom > dateTo); 404 — аналог не найден или пустой artikul.

## Интерпретация данных

- **sales** — продажи за день в штуках (остаток на конец предыдущего дня минус остаток на конец текущего; при отсутствии предыдущего дня или при росте остатка — 0).
- **revenue** — выручка за день (sales × price на эту дату).
- **isDeliveryDay** — день поставки: остаток вырос по сравнению с предыдущим днём; в такой день sales = 0, на графике можно подсветить точку или подпись.

Оба эндпоинта требуют авторизации (checkAuth + роль USER). Передавать JWT в заголовке `Authorization: Bearer <token>`.
