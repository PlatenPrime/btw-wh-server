# Фронтенд: внеочередной compensating slice

## Задача

Дать админу кнопку «компенсировать срез» с диалогом выбора конкурента. После подтверждения вызвать API и показать результат (сколько позиций опросили / обновили). Это **не** полный пересбор каталога — только повторный опрос «битых» остатков/цен за **сегодня**.

## API

- Метод/путь: `POST /api/slice-compensation/run`
- Auth: JWT, роль ≥ ADMIN
- Тело: `{ "konkName": string }`
- Полный контракт ответов: [`docs/api/slice-compensation.md`](../api/slice-compensation.md)

Список конкурентов для селекта — из существующего `GET /api/konks` (или уже закэшированного справочника на клиенте). Имя в запросе можно слать в любом регистре; бэкенд нормализует.

## UX

### Точка входа

Кнопка рядом со срезами / конкурентом (доступна только ADMIN/PRIME). Подпись в духе «Компенсирующий срез» / «Повторить опрос».

### Диалог

1. Выбор конкурента (`konkName`) — обязателен; без выбора Submit неактивен.
2. Короткое пояснение: обновятся только позиции с ошибкой опроса в сегодняшних analog- и sku-срезах; запрос может идти долго (последовательный scrape).
3. Confirm / Cancel.
4. После Confirm — loading на диалоге или кнопке; повторный Confirm того же конкурента не слать, пока предыдущий запрос в полёте (на бэке всё равно вернётся 409).

### После ответа

| Статус | Поведение UI |
|--------|----------------|
| 200 | Закрыть диалог или оставить summary: `analog.refetched/updated`, `sku.refetched/updated`, дата `sliceDate`; при необходимости инвалидировать кэш срезов за этот день |
| 400 | Показать ошибку валидации (обычно пустой `konkName`) |
| 409 | Toast: компенсация по этому конкуренту уже выполняется |
| 401/403 | Стандартная обработка сессии/прав |
| 500 | Toast ошибки; предложить повторить позже |

### Не делать

- Не вызывать primary slice cron-утилиты через другие эндпоинты — их нет.
- Не обещать мгновенный ответ: тайм아웃 клиента должен быть достаточным для длинного scrape (минуты при большом числе `-1`).
- Не трактовать `refetched > 0 && updated === 0` как полный провал: опрос прошёл, но данные снова пришли как `-1/-1` (сайты/прокси).
