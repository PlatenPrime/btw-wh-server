## Удаление полей `title` и `imageUrl` из Analog

### Краткое резюме

- **Что изменилось**: из модели аналога (`Analog`) полностью удалены поля `title` и `imageUrl`. Они больше не хранятся в базе и не участвуют в работе эндпоинтов.
- **Где актуально**: все эндпоинты `/api/analogs` (создание, обновление, выборки, получение по id).
- **Причина**: в прод‑данных эти поля не использовались, а основным источником имени/картинки остаются сущности `Prod` и `Konk` (и связанные модули остатков конкурентов).

### Влияние на API

- **POST `/api/analogs`**
  - Раньше: в теле могли быть поля `title` и `imageUrl`; при пустом `artikul` они были обязательными.
  - Теперь:
    - Тело запроса **больше не принимает** `title` и `imageUrl`.
    - Обязательные поля: `konkName`, `prodName`, `url`.
    - Поле `artikul`:
      - если не указано или пустое — запрос считается невалидным (400).
      - если указано — при создании подтягивается `nameukr` из `Art` (если документ найден).

- **PATCH `/api/analogs/id/:id`**
  - Раньше: можно было обновлять `title` и `imageUrl`.
  - Теперь:
    - Разрешено обновлять только: `konkName`, `prodName`, `artikul`, `nameukr`, `url`.
    - Поля `title` и `imageUrl` в теле запроса **игнорируются схемой валидации** (и их не нужно отправлять).

- **GET эндпоинты `/api/analogs`**
  - **GET `/api/analogs`**, **`/api/analogs/prod/:prodName`**, **`/api/analogs/konk/:konkName`**, **`/api/analogs/artikul/:artikul`**:
    - В массиве `analogs` объекты теперь содержат только:
      - `_id`
      - `konkName`
      - `prodName`
      - `artikul`
      - `nameukr?`
      - `url`
      - `createdAt?`
      - `updatedAt?`
    - Полей `title` и `imageUrl` **нет**.
  - **GET `/api/analogs/id/:id`**:
    - Сам объект аналога не содержит `title`/`imageUrl`.
    - Как и раньше, дополнительно возвращаются:
      - `konk: { id, name, title, imageUrl }` — данные конкурента
      - `prod: { id, name, title, imageUrl }` — данные производителя
    - Эти поля относятся к `Konk` и `Prod` и **не изменялись**.

### Что нужно сделать на фронтенде

- **Формы создания/редактирования аналога**
  - Удалить поля ввода `title` и `imageUrl`, если они были.
  - Гарантировать, что при создании/редактировании всегда передаётся непустой `artikul`.
  - Если ранее учитывались ошибки вида _“When artikul is empty, title and imageUrl are required”_, заменить обработку на новое сообщение об ошибке по `artikul` (400, invalid/empty artikul).

- **Таблицы/списки аналогов**
  - Если где‑то использовались `analog.title` или `analog.imageUrl` (например, колонка “Название аналога” или превью картинки):
    - Заменить:
      - название на `analog.nameukr` (если нужно имя из Art) или на комбинацию из `prod.title`/`konk.title`, если данные берутся через связанный API;
      - картинку — на `prod.imageUrl` или `konk.imageUrl`, если уже есть соответствующие данные, либо убрать колонку.

- **Поиск/фильтрация**
  - Поиск по аналогам больше не использует поле `title` аналога:
    - утилита фильтрации теперь учитывает только `nameukr`.
  - Если в UI был фильтр “по названию аналога” — его нужно синхронизировать с текущей логикой (поиск по `nameukr`).

### Совместимость и миграция

- В базе **нет** документов с заполненными `title`/`imageUrl`, поэтому:
  - дополнительных миграций данных не требуется;
  - проблем с чтением старых документов быть не должно.
- Фронтенду важно:
  - перестать опираться на `Analog.title` и `Analog.imageUrl`;
  - использовать данные `Prod`/`Konk` и/или `Art` там, где нужны имя и картинка товара.
