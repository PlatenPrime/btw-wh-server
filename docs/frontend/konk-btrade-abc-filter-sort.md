# Фильтр и сортировка по ABC для эндпоинтов konk-btrade (фронтенд)

Документация для фронтенда: опциональные параметры `abc` и `sortBy` в групповых эндпоинтах сравнения конкурента и Btrade по конкуренту, производителю и периоду дат.

## Затрагиваемые эндпоинты

Все четыре эндпоинта принимают одни и те же опциональные query-параметры:

| Метод и путь | Ответ |
|--------------|--------|
| `GET /api/analog-slices/konk-btrade/comparison-excel` | Excel (.xlsx) |
| `GET /api/analog-slices/konk-btrade/sales-comparison-excel` | Excel (.xlsx) |
| `GET /api/analog-slices/konk-btrade/sales-comparison` | JSON |
| `GET /api/analog-slices/konk-btrade/stock-comparison` | JSON |

Обязательные параметры для всех: `konk`, `prod`, `dateFrom`, `dateTo` (как и раньше).

## Параметр `abc` (опционально)

- **Тип:** строка, одна из букв: `"A"`, `"B"`, `"C"`, `"D"`.
- **Семантика:** фильтрация по полю ABC артикула. В выборку попадают только те артикулы, у которых в поле ABC (из справочника артикулов) присутствует переданная буква. Сравнение без учёта регистра (например, `abc=B` подходит для артикула с ABC `"101B"` или `"50b"`).
- **Если не передан:** фильтр не применяется — возвращаются все аналоги по заданным `konk`, `prod` и периоду (текущее поведение).

Пример: артикул с ABC `101B` попадёт в выборку только если `abc` не передан или передан `abc=B`.

## Параметр `sortBy` (опционально)

- **Тип:** строка, допустимое значение: `"abc"`.
- **Семантика:** при **одновременной** передаче `abc` и `sortBy=abc` порядок строк (аналогов) в ответе — по числовой части поля ABC. Число берётся из начала строки ABC (например, из `"101B"` — число `101`). Сортировка по этому числу по возрастанию; при равенстве — по артикулу.
- **Если передан только `sortBy=abc` без `abc`:** сортировка по ABC не применяется, порядок по артикулу.
- **Если `sortBy` не передан:** порядок по артикулу (по возрастанию).

## Примеры запросов

- Без фильтра и без сортировки (как раньше):
  ```
  GET /api/analog-slices/konk-btrade/sales-comparison?konk=air&prod=gemar&dateFrom=2026-03-01&dateTo=2026-03-31
  ```

- Только фильтр по букве B (порядок по артикулу):
  ```
  GET /api/analog-slices/konk-btrade/sales-comparison?konk=air&prod=gemar&dateFrom=2026-03-01&dateTo=2026-03-31&abc=B
  ```

- Фильтр по B и сортировка по числовой части ABC:
  ```
  GET /api/analog-slices/konk-btrade/sales-comparison?konk=air&prod=gemar&dateFrom=2026-03-01&dateTo=2026-03-31&abc=B&sortBy=abc
  ```

- Скачивание Excel с фильтром по A и сортировкой по ABC:
  ```
  GET /api/analog-slices/konk-btrade/comparison-excel?konk=air&prod=gemar&dateFrom=2026-03-01&dateTo=2026-03-31&abc=A&sortBy=abc
  ```

## Валидация

- `abc` допускает только значения `"A"`, `"B"`, `"C"`, `"D"` (одна буква). Иные значения приводят к ответу **400** с телом `{ "message": "Validation error", "errors": [...] }`.
- `sortBy` допускает только значение `"abc"`. Иные значения — **400** (Validation error).

## Итоги и пустой результат

- Все итоги (в JSON: `days`, `summary`; в Excel: блоки «ВСЬОГО», итоговая секция) считаются **только по отфильтрованным данным**. То есть при переданном `abc` итоги соответствуют только артикулам, прошедшим фильтр по букве ABC.
- Если после применения фильтра `abc` не осталось ни одного аналога, все четыре эндпоинта возвращают **404** с телом `{ "message": "Analogs not found for provided konk/prod" }` (то же сообщение, что и при отсутствии аналогов по `konk`/`prod` без фильтра).

## Сводка для UI

- Добавить в формы запросов к этим эндпоинтам опциональные поля:
  - **ABC (фильтр):** выбор одной буквы A / B / C / D или «все».
  - **Сортировка:** опция «по артикулу» (по умолчанию) или «по ABC» (доступна только при выбранном фильтре ABC).
- При «по ABC» передавать `sortBy=abc` вместе с выбранным `abc`.
- Обрабатывать 400 при неверных значениях и 404 при пустом наборе после фильтра.
