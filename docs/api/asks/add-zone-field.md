# Добавление поля zone в заявки (Ask)

## Описание изменения

В API создания заявок добавлено опциональное поле `zone` (строка), которое позволяет указать зону склада при создании заявки.

## Обратная совместимость

Изменение полностью обратно совместимо. Поле `zone` является опциональным, поэтому существующий код продолжит работать без изменений.

## API изменения

### POST `/api/asks`

**Обновленный формат запроса:**

```typescript
{
  artikul: string;
  nameukr?: string;
  quant?: number;
  com?: string;
  sklad?: "pogrebi" | "merezhi";
  zone?: string;        // ← Новое опциональное поле
  askerId: string;
}
```

**Пример запроса:**

```json
{
  "artikul": "ART-001",
  "nameukr": "Товар",
  "quant": 5,
  "com": "Комментарий",
  "zone": "Зона А",
  "askerId": "507f1f77bcf86cd799439011"
}
```

**Пример ответа (201):**

```json
{
  "_id": "507f1f77bcf86cd799439012",
  "artikul": "ART-001",
  "nameukr": "Товар",
  "quant": 5,
  "com": "Комментарий",
  "sklad": "pogrebi",
  "zone": "Зона А",
  "status": "new",
  "asker": "507f1f77bcf86cd799439011",
  "askerData": {
    "_id": "507f1f77bcf86cd799439011",
    "fullname": "Иван Иванов"
  },
  "actions": [],
  "pullQuant": 0,

  "events": [...],
  "createdAt": "2025-01-15T10:00:00.000Z",
  "updatedAt": "2025-01-15T10:00:00.000Z"
}
```

## Формат данных Ask

Поле `zone` теперь доступно во всех ответах API, которые возвращают объект `Ask`:

```typescript
{
  // ... другие поля
  sklad?: string;
  zone?: string;  // ← Новое опциональное поле
  // ... остальные поля
}
```

## Примечания

- Поле `zone` опциональное и может быть не указано при создании заявки
- Если поле не указано, оно будет `undefined` в ответе API
- Поле может быть использовано для фильтрации или отображения заявок по зонам склада
