# Изменение формата artikuls в поставках (для фронтенда)

## Что изменилось

В полном документе поставки (Del) поле `artikuls` больше не имеет вид «артикул → число». Теперь по каждому артикулу хранится объект:

```ts
artikuls: {
  [artikul: string]: {
    quantity: number;   // количество
    nameukr?: string;   // название (укр.), опционально
  };
}
```

## Где это видно

- **GET `/api/dels/id/:id`** — в `data.artikuls` каждый артикул — объект с `quantity` и при необходимости `nameukr`.
- **PATCH `/api/dels/:id/artikuls/:artikul`** — в ответе обновлённый документ с тем же форматом.
- **POST `/api/dels/:id/artikuls/update-all`** — фоновое обновление; после завершения при следующем GET по id поставки `artikuls` уже в новом формате.

## Создание поставки (без изменений для клиента)

**POST `/api/dels`** — тело запроса не меняется: по-прежнему передаётся объект «артикул → количество»:

```json
{ "title": "Поставка 1", "artikuls": { "ART-001": 10, "ART-002": 5 } }
```

Сервер сам подставляет `nameukr` из справочника артикулов (arts); для неизвестных артикулов поле не задаётся. В ответе `data` уже содержит новый формат `artikuls`.

## Что сделать на фронте

1. Типы/интерфейсы для Del: заменить `Record<string, number>` на объект с `quantity` и опциональным `nameukr` по каждому артикулу.
2. Отображение списка артикулов поставки: брать количество из `artikuls[artikul].quantity`, название — из `artikuls[artikul].nameukr` (если есть).
