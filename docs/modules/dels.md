# Модуль Dels (Поставки)

## Описание модуля

Модуль `dels` решает задачи учёта поставок. Он обеспечивает хранение поставок с привязкой к артикулам, разделение количества в поставке и остатка на sharik.ua, обновление остатков из внешнего источника, а также получение списка поставок без тяжёлых данных об артикулах для быстрого отображения на фронтенде.

## Сущности модуля

### Del (Поставка)

Поставка представляет собой документ с названием (`title`), обязательной привязкой к производителю (`prodName` — соответствует полю `name` в модели Prod), опциональным вложенным объектом `prod` с полями `title` и `imageUrl` из справочника Prod (заполняется при создании и при PATCH title), датами создания и обновления, а также объектом `artikuls`: ключи — строки-артикулы (как в моделях Art, Def, Pos), значения — объекты `{ quant: number, stock?: number, nameukr?: string }`.

- **quant** — количество товара в поставке; задаётся клиентом при создании (`POST /api/dels`).
- **stock** — остаток на sharik.ua; заполняется при обновлении артикулов из sharik (PATCH одного артикула, POST update-all). До первого sync может отсутствовать.
- **nameukr** — подставляется из коллекции arts при создании; при sharik-update обновляется из ответа getSharikStockData.

## Связи между сущностями

- **Del → Prod**: Поставка привязана к производителю через поле `prodName`, равное `Prod.name`. В поддокументе `prod` сохраняются `title` и `imageUrl` из Prod при создании и при обновлении заголовка (PATCH title). Явной ссылки (ObjectId) в схеме нет; валидация при создании и обновлении проверяет существование записи в Prod.
- **Del → Art / Def / Pos**: Косвенная связь по смыслу. Ключи в `artikuls` совпадают с полем `artikul` в артикулах, дефицитах и позициях. Явных ссылок в схеме нет.

## Концепции и принятые решения

### quant и stock

Количество в поставке (`quant`) и остаток на sharik (`stock`) разделены: клиент при создании передаёт только количество в поставке; остаток подтягивается отдельными операциями sync с sharik.ua. При обновлении из sharik `quant` не перезаписывается.

### Обновление значений из sharik.ua

Остатки (`stock`) артикулов в поставке можно обновлять данными с sharik.ua (getSharikStockData). Поддерживается обновление одного артикула по id поставки и артикулу, а также фоновое обновление всех артикулов поставки. При фоновом обновлении ответ возвращается сразу (202), процесс выполняется асинхронно.

### Список без artikuls

Эндпоинт получения списка поставок возвращает `_id`, `title`, `prodName`, `prod` (если есть), `createdAt`, `updatedAt`, без поля `artikuls`, чтобы уменьшить объём данных при отображении списка.

### Удаление только для PRIME

Удаление поставки разрешено только роли PRIME, остальные операции создания и обновления — ADMIN и выше, чтение — USER и выше.

### Миграция legacy-данных

Документы со старым полем `quantity` (остаток sharik) мигрируются скриптом: `quantity` → `stock`, `quant = 0`. См. `src/modules/dels/scripts/runMigrateDelArtikulsQuantStock.ts`.

## Форматы данных

### Del (полный документ)

- `_id`: string (MongoDB ObjectId)
- `title`: string (название поставки)
- `prodName`: string (производитель, соответствует Prod.name)
- `prod`: объект (опционально): `{ title: string, imageUrl: string }` — данные производителя из Prod
- `artikuls`: объект «артикул → { quant, stock?, nameukr? }»
- `createdAt`, `updatedAt`: Date

### Элемент списка (без artikuls)

- `_id`: string
- `title`: string
- `prodName`: string
- `prod`: объект (опционально): `{ title: string, imageUrl: string }`
- `createdAt`, `updatedAt`: Date
