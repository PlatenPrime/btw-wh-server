# Модуль Kasks

## Описание модуля

Модуль **kasks** объединяет идеи «каса» и «asks»: это **запросы доставить товар к кассе**. В отличие от модуля `asks`, здесь нет статусов, pull, asker/solver и жизненного цикла заявки — только фиксация запроса с артикулом, названием, количеством, зоной и комментарием. Удобно для сценариев, когда нужно уведомить кассу/склад о необходимости подвести товар без полного workflow заявки.

## Сущность Kask

| Поле    | Тип    | Описание                          |
| ------- | ------ | --------------------------------- |
| artikul | string | Артикул                           |
| nameukr | string | Название украинском               |
| quant   | number | Количество                        |
| zone    | string | Зона                              |
| com     | string | Комментарий                       |

Все перечисленные поля **задаются клиентом при создании** и хранятся в документе. Дополнительно Mongoose добавляет `createdAt` и `updatedAt` (timestamps), по ним строится выборка **по дате** (`getKasksByDate`).

## Уведомления через Telegram

При создании Kask отправляется сообщение в **чат кассы** `@kassabtw` (`sendMessageToKasaChat`). В сообщении — артикул, название, количество с « шт», зона, комментарий и ссылка на картинку на sharik.ua по артикулу (`.../elements_big/{artikul}_m1.jpg`). Для пользователей с ролью **PRIME** и в окружении **test** отправка не выполняется — по аналогии с уведомлениями asks.

## API эндпоинты

Подробное описание запросов и ответов — в [API Kasks](../api/kasks.md).

- **POST `/api/kasks`** — создать запрос к кассе.
- **GET `/api/kasks/by-date`** — список за дату.
- **GET `/api/kasks/:id`** — один документ по id.
- **PATCH `/api/kasks/:id`** — частичное обновление.
- **DELETE `/api/kasks/:id`** — удаление.

Доступ ко всем операциям: авторизованный пользователь с ролью **USER** (без проверки владельца записи).
