# Модуль Skugrs (группы товаров конкурента)

## Описание модуля

Модуль `skugrs` хранит **группы товаров конкурента**: логические наборы карточек SKU (`Sku`) для заданного конкурента (`konkName`) и производителя (`prodName`). У одной пары конкурент + производитель может быть несколько групп (например, разные размеры или подборки).

## Сущности модуля

### Skugr (группа товаров конкурента)

- `konkName`: string — имя-ключ конкурента
- `prodName`: string — имя-ключ производителя
- `title`: string — название группы
- `url`: string — ссылка на страницу сайта конкурента с этой группой товаров
- `skus`: массив MongoDB ObjectId — ссылки на документы коллекции `skus`
- `_id`, `createdAt`, `updatedAt`: системные поля

## Связи между сущностями

**Skugr → Sku**: группа содержит массив идентификаторов товаров конкурента. Это основная структурная связка модуля. Обратной ссылки от `Sku` к группе нет: удаление группы не удаляет документы SKU.

## Концепции и принятые решения

### Состав группы при создании

Массив `skus` задаётся в теле `POST /api/skugrs`. Дубликаты id в запросе приводятся к уникальному набору перед сохранением. Все переданные id должны существовать в коллекции `skus`, иначе возвращается ошибка 400.

### Изменение состава `skus` после создания

Отдельный API для добавления/удаления SKU в группе в текущей версии **не** реализован; планируется позже.

### Редактирование метаданных

Через `PATCH` можно менять только `konkName`, `prodName`, `title`, `url`.

### Индексы

Составной индекс `{ konkName: 1, prodName: 1 }` ускоряет выборки по конкуренту и производителю.

### Роли доступа

Как у модуля `skus`:

- чтение (`GET`) — роль USER;
- создание и обновление (`POST`, `PATCH`) — роль ADMIN;
- удаление (`DELETE`) — роль PRIME.

## API эндпоинты

- `GET /api/skugrs` — список групп с фильтрами и пагинацией
- `POST /api/skugrs` — создание группы
- `PATCH /api/skugrs/id/:id` — обновление полей группы
- `DELETE /api/skugrs/id/:id` — удаление группы

Подробности: [API Skugrs](../api/skugrs.md).

## Формат данных в ответах API

Поле `_id` и элементы `skus` отдаются строками (hex ObjectId). Остальные поля — как в модели.
