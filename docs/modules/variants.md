# Модуль Variants (Варианты)

## Описание модуля
Модуль `variants` хранит записи вариантов товаров на сайтах/страницах конкурентов (в отличие от `analogs`, варианты не привязаны к `artikul` и не содержат поля `nameukr`). Модуль поддерживает CRUD-операции, а также чтение со встроенной пагинацией и фильтрацией.

## Сущности модуля

### Variant (Вариант)
Документ варианта содержит:
- `konkName` — ключ конкурента (соответствует `name` в модели `Konk`);
- `prodName` — ключ производителя (соответствует `name` в модели `Prod`);
- `title` — название варианта;
- `url` — ссылка на страницу варианта (уникальна);
- `varGroup?` — опциональная группа вариантов в формате объекта `{ id, title }`;
- `imageUrl` — ссылка на изображение варианта;
- стандартные временные метки `createdAt`/`updatedAt`.

## Принятые решения

### Поиск и сортировка
Список вариантов (`GET /api/variants`) поддерживает поиск по полю `title` (регистронезависимо) и сортировку по `title` (по возрастанию).

### Роли доступа
- чтение (все `GET`) — `checkAuth` + `checkRoles(USER)`;
- создание и обновление — `checkAuth` + `checkRoles(ADMIN)`;
- удаление — `checkAuth` + `checkRoles(PRIME)`.

## API эндпоинты

- `GET /api/variants` — список вариантов с пагинацией и фильтрами (конкурент/производитель, поиск по title);
- `GET /api/variants/id/:id` — получение варианта по id;
- `POST /api/variants` — создание варианта;
- `PATCH /api/variants/id/:id` — обновление варианта по id;
- `DELETE /api/variants/id/:id` — удаление варианта по id.

Детальное описание запросов и ответов — в [API документации](../api/variants.md).
