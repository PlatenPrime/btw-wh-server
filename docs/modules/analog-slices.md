# Модуль Analog Slices (Срезы аналогов по датам)

## Описание модуля

Модуль `analog-slices` хранит ежедневные срезы остатков (stock) и цен (price) аналогов по конкурентам. Один документ среза соответствует паре «конкурент + дата» и содержит объект `data`, в котором ключи — артикулы (artikul) аналогов, значения — объекты `{ stock, price }`. Срезы формируются по расписанию (cron) путём опроса данных по каждому аналогу конкурента; для выборки по конкретному аналогу и дате или периоду используются контроллеры чтения.

## Сущности модуля

### AnalogSlice (Срез аналогов)

Документ ежедневного среза по одному конкуренту. Поля: `konkName` — ключ конкурента (соответствует полю `name` в модели Konk и полю `konkName` в Analog); `date` — дата среза (начало дня UTC); `data` — объект, ключи которого артикулы аналогов, значения — `{ stock: number, price: number, artikul?: string }`. Уникальный индекс: `(konkName, date)`.

## Связи между сущностями

- **Analog:** по `konkName` и артикулу (ключ в `data`) срез связан с аналогами. Для получения данных по одному аналогу используется его `_id`: по нему загружается аналог, из него берутся `konkName` и `artikul` для доступа к `data[artikul]` в документах срезов. У аналогов с пустым `artikul` записей в срезах нет — для них эндпоинты по аналогу возвращают 404.

## Концепции и принятые решения

### Нормализация даты

Даты срезов хранятся и запрашиваются как начало дня по UTC (например, через `toSliceDate` в утилитах), чтобы единообразно сопоставлять запросы с сохранёнными документами.

### Эндпоинты по аналогу и группе аналогов

- **Срез на одну дату:** по `analogId` (path) и `date` (query YYYY-MM-DD) возвращается одна точка `{ stock, price }`. Используется для карточки товара на конкретную дату.
- **Срез за период:** по `analogId` (path) и `dateFrom`, `dateTo` (query YYYY-MM-DD) возвращается массив `{ date, stock, price }[]`, отсортированный по дате. Формат удобен для построения графиков (в т.ч. Recharts / shadcn/ui).
- **Продажи на одну дату:** по `analogId` (path) и `date` (query YYYY-MM-DD) возвращается одна точка `{ sales, revenue, price, isDeliveryDay }`. Используется для графиков и карточек по продажам за день.
- **Продажи за период:** по `analogId` (path) и `dateFrom`, `dateTo` (query YYYY-MM-DD) возвращается массив `{ date, sales, revenue, price, isDeliveryDay }[]`, отсортированный по дате. Формат удобен для построения графиков продаж и выручки.
- **Экспорт Excel сравнения аналога и Btrade:** по `analogId` (path) и `dateFrom`, `dateTo` (query) возвращается файл .xlsx с таблицей: остатки и цены аналога (конкурент) и Btrade по датам, колонки «Різниця» и «Різниця, %», а также обобщающие колонки по динамике остатков Btrade vs конкурент.
- **Групповой Excel-отчёт по конкуренту и производителю:** по паре `konk` + `prod` (query) и периоду `dateFrom`, `dateTo` (query YYYY-MM-DD) формируется один файл .xlsx, в котором для каждого аналога этой группы строится такой же 4-строчный блок (остатки/цены аналога и Btrade), как в одиночном отчёте. Блоки идут подряд и отсортированы по артикулу по возрастанию. Отчёт предназначен для аналитики конкурента по бренду/производителю.
- **Экспорт Excel сравнения продаж и выручки (один аналог):** по `analogId` (path) и `dateFrom`, `dateTo` (query) возвращается файл .xlsx с таблицей по продажам (разница остатка с предыдущим днём), ценам и выручке (продажи × цена) аналога и Btrade по датам; блок 6 строк, колонка «Всього», четыре колонки дельт (Δ Продажі шт/%, Δ Виручка грн/%); внизу — итоговая секция ключ–значение.
- **Экспорт Excel сравнения продаж и выручки (группа аналогов):** по паре `konk` + `prod` (query) и периоду `dateFrom`, `dateTo` формируется один файл .xlsx с блоками по 6 строк на каждый аналог (продажи/ціна/виручка аналога и Btrade), колонками «Всього» и четырьмя дельтами; внизу — итоговая секция ключ–значение. Логика колонок дельт аналогична отчёту по остаткам: разница в штуках/процентах для продаж и в гривнах/процентах для выручки; значения записываются в первую строку блока и отображаются в объединённых ячейках.

## Роли доступа

Все эндпоинты модуля доступны только для чтения и требуют роль USER (checkAuth + checkRoles(USER)).

## API эндпоинты

- **GET `/api/analog-slices`** — срез по конкуренту и дате (query: `konkName`, `date`). Возвращает весь объект среза (konkName, date, data). Доступ: USER.
- **GET `/api/analog-slices/analog/:analogId`** — срез по конкретному аналогу на одну дату (query: `date`). Ответ: `{ stock, price }`. Доступ: USER.
- **GET `/api/analog-slices/analog/:analogId/range`** — данные по аналогу за период дат (query: `dateFrom`, `dateTo`). Ответ: массив `{ date, stock, price }[]`. Доступ: USER.
- **GET `/api/analog-slices/analog/:analogId/sales-by-date`** — продажи и выручка по аналогу на одну дату (query: `date`). Ответ: `{ sales, revenue, price, isDeliveryDay }`. Доступ: USER. См. [документацию для фронтенда](../frontend/analog-sales-charts.md).
- **GET `/api/analog-slices/analog/:analogId/sales-range`** — продажи и выручка по аналогу за период (query: `dateFrom`, `dateTo`). Ответ: массив `{ date, sales, revenue, price, isDeliveryDay }[]`. Доступ: USER. См. [документацию для фронтенда](../frontend/analog-sales-charts.md).
- **GET `/api/analog-slices/analog/:analogId/comparison-excel`** — экспорт Excel с сравнением срезов аналога и Btrade за период (query: `dateFrom`, `dateTo`). Ответ: файл .xlsx. Доступ: см. [API](../api/analog-slices.md) (сейчас без проверки роли; при включении checkAuth — USER).
- **GET `/api/analog-slices/konk-btrade/comparison-excel`** — экспорт Excel с сравнением срезов Btrade и конкурента сразу по группе аналогов, отфильтрованных по `konk` и `prod` за период (query: `konk`, `prod`, `dateFrom`, `dateTo`). Ответ: файл .xlsx. Доступ: см. [API](../api/analog-slices.md).
- **GET `/api/analog-slices/analog/:analogId/sales-comparison-excel`** — экспорт Excel с сравнением продаж и выручки аналога и Btrade за период (query: `dateFrom`, `dateTo`). Ответ: файл .xlsx. Доступ: см. [API](../api/analog-slices.md).
- **GET `/api/analog-slices/konk-btrade/sales-comparison-excel`** — экспорт Excel с сравнением продаж и выручки по группе аналогов (query: `konk`, `prod`, `dateFrom`, `dateTo`). Ответ: файл .xlsx. Доступ: см. [API](../api/analog-slices.md).
- **GET `/api/analog-slices/konk-btrade/sales-comparison`** — агрегированные продажи и выручка конкурента vs Btrade по дням за период (query: `konk`, `prod`, `dateFrom`, `dateTo`). Данные суммарные по всем артикулам (не детализированные) — для построения графиков на фронтенде. Включает ежедневные показатели и итоговую секцию. Ответ: JSON. Доступ: USER. См. [документацию для фронтенда](../frontend/konk-btrade-sales-comparison.md).

Детальное описание запросов и ответов — в [API документации](../api/analog-slices.md). Документация для фронтенда по эндпоинтам продаж для графиков — [../frontend/analog-sales-charts.md](../frontend/analog-sales-charts.md), [../frontend/konk-btrade-sales-comparison.md](../frontend/konk-btrade-sales-comparison.md).
