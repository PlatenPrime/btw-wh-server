# Модуль SKU Slices (Срезы SKU по датам)

## Назначение

Модуль хранит ежедневные срезы остатков и цен по товарам конкурентов, представленным сущностью SKU. Один документ среза соответствует паре «конкурент (`konkName`) + календарная дата» и содержит объект `data`, в котором ключи — канонические идентификаторы товаров (`Sku.productId`, формат `{konkName в нижнем регистре}-{сырой id с сайта конкурента}`), значения — остаток и цена, полученные тем же способом скрапинга страницы товара, что используется для аналогов (по URL SKU).

Модуль не использует данные Btrade: нет сравнений, агрегатов и отчётов, опирающихся на артикулы Btrade. Excel-выгрузки содержат только поля идентификации из SKU и числовые ряды по датам из срезов конкурента. Для срезовых Excel используются метрики `Залишок` и `Ціна`, для sales Excel используются `Продажі`, `Ціна`, `Виручка`.

## Сущности

### SkuSlice

Документ дневного среза по одному конкуренту. Поля: `konkName`, `date` (начало суток по UTC), `data` — вложенный объект с ключами `productId` и значениями `{ stock, price }`. Уникальный составной индекс по `(konkName, date)`.

## Связи

- **Sku:** каждая запись в `data` соответствует одному SKU с тем же `productId` и тем же `konkName`, что и документ среза. Доступ к точке среза по конкретному товару на фронте обычно идёт через `skuId`: сервер загружает SKU, читает `productId` и `konkName`, затем ищет значение в документе среза.

- **Парсеры групп конкурентов:** при заполнении SKU из обхода страниц группы в документ записывается `productId`, согласованный с правилом канонизации (склейка нормализованного имени конкурента и id товара со страницы). Это обеспечивает стабильные ключи в `data` без кодирования URL.

## Сбор данных

По расписанию (cron) для каждого конкурента, по которому в базе есть хотя бы один SKU, запускается сбор среза на текущую календарную дату. Конкуренты обрабатываются параллельно; внутри одного конкурента SKU обходятся последовательно с фиксированной паузой между запросами к сайту конкурента. Неподдерживаемый для скрапинга конкурент приводит к прерыванию среза по этому конкуренту после первой же ошибки типа «неподдерживаемый konk»; прочие ошибки по отдельным SKU логируются, остальные позиции продолжают обрабатываться.

Часовой пояс cron: Европа/Киев, момент запуска — полночь.

## HTTP и отчёты

Предусмотрены только операции чтения JSON (срез по конкуренту и дате, ряды по одному SKU, продажи и выручка по тем же правилам, что и для срезов аналогов) и четыре Excel-выгрузки: по одному SKU (остатки), по группе SKU с фильтром конкурент + производитель (остатки), по одному SKU (продажи), по группе SKU конкурента (продажи). Заголовки колонок метаданных в Excel на украинском языке и отражают поля SKU (назва, посилання, ідентифікатор, конкурент, виробник).

Для групповой выгрузки (`/api/sku-slices/konk/excel`) после всех блоков SKU добавляется строка `Підсумок` с агрегатами по колонкам `Різниця` и `Різниця, %`.

## Роли

Эндпоинты модуля ориентированы на чтение с аутентификацией и ролью USER, аналогично JSON-роутам модуля срезов аналогов.

## API эндпоинты

- **GET `/api/sku-slices`** — срез по `konkName` и `date`.
- **GET `/api/sku-slices/sku/:skuId`** — точка среза на дату (query `date`).
- **GET `/api/sku-slices/sku/:skuId/range`** — срез за период (`dateFrom`, `dateTo`).
- **GET `/api/sku-slices/sku/:skuId/sales-by-date`** и **sales-range** — продажи и выручка на дату и за период.
- **GET `/api/sku-slices/sku/:skuId/slice-excel`** — Excel по одному SKU за период.
- **GET `/api/sku-slices/konk/excel`** — Excel по группе SKU (`konk`, `prod`, период).
- **GET `/api/sku-slices/sku/:skuId/sales-excel`** — Excel продаж по одному SKU за период.
- **GET `/api/sku-slices/konk/sales-excel`** — Excel продаж по SKU группы `konk` + `prod` за период.

Подробные форматы запросов и ответов — в [API документации](../api/sku-slices.md).
